# 🛠️ OpenClaw 설치 인터랙티브 가이드 (INTERACTIVE_GUIDE.md)

이 문서는 사용자가 직접 커맨드를 실행하며 OpenClaw 환경을 구축할 수 있도록 돕는 실시간 가이드이다. 각 단계가 완료될 때마다 상태를 갱신한다.

---

### 📋 전체 진행 로드맵

| 단계 | 목표 | 상태 | 주요 커맨드 |
| :--- | :--- | :--- | :--- |
| **1단계** | 설치 전 진단 및 공식 문서 동기화 | 🔄 대기 중 | `/kb:sys-stat`, `/kb:oc-update setup` |
| **2단계** | 환경 스냅샷 생성 | ⏳ 대기 | `env > snapshot.log` 등 |
| **3단계** | OpenClaw CLI 설치 및 검증 | ⏳ 대기 | `npm install -g openclaw@latest` |
| **4단계** | 최종 보고 및 온보딩 준비 | ⏳ 대기 | `openclaw doctor` |

---

### 🎯 현재 진행: 1단계 - 설치 전 진단 및 공식 문서 동기화

#### 1. 실행할 커맨드
아래 커맨드를 순차적으로 실행해 주세요.
- `[1] /kb:sys-stat` : 현재 `jarvis` 계정의 Node.js 버전 및 시스템 자원을 체크합니다.
- `[2] /kb:oc-update setup` : 공식 사이트의 최신 설치 명세를 가져옵니다.

#### 💡 검토 및 결정 사항
- **Node.js 버전**: 공식 문서에서 권장하는 버전(v22.14+ 또는 v24)과 현재 시스템 버전이 일치하는지 확인이 필요합니다.
- **공식 명세 변경**: `CORE_SPEC.md`에 기록된 이전 명세와 새로 수집된 명세 사이에 '파괴적 변경(Breaking Changes)'이 있는지 확인 후 진행 여부를 결정해야 합니다.
