### 1. 목적 (Objective)
에이전트의 실행 권한을 작업의 위험도에 따라 분리하여, 사용자의 호스트 시스템 보안을 유지하면서도 업무 효율성을 극대화한다.

---

### 2. 실행 구역 분류 (Execution Zones)

에이전트는 모든 태스크를 아래 세 가지 구역 중 하나로 분류한 뒤 실행해야 한다.

| 구역 명칭 | 위험도 | 실행 환경 | 허용 범위 |
| :--- | :--- | :--- | :--- |
| **Zone 0: Core** | 낮음 | 호스트 (Host) | 로컬 파일 읽기/쓰기, Git 조작, 시스템 정보 조회 |
| **Zone 1: Action** | 중간 | 호스트 (Specific) | 이미 로그인된 브라우저 제어 (카톡, KTX 등), 신뢰된 API 호출 |
| **Zone 2: Sandbox** | 높음 | **Docker/Micro-VM** | 외부 코드 실행, 미검증 웹 크롤링, 패키지 설치 (`npm install` 등) |

---

### 3. 구역 결정 규칙 (Decision Logic)

에이전트는 태스크를 받으면 다음 체크리스트를 수행한다.
1.  **외부 코드를 실행하는가?** → Yes: **Zone 2**
2.  **인터넷에서 가져온 데이터를 파싱/실행하는가?** → Yes: **Zone 2**
3.  **사용자의 개인 세션(로그인)이 필요한가?** → Yes: **Zone 1**
4.  **단순 지식 노동 및 로컬 파일 정리인가?** → Yes: **Zone 0**

---

### 4. 데이터 반출입 및 통신 규정 (Data I/O)

* **샌드박스 진입**: 모든 입력 데이터는 `root/temp_in/` 폴더를 통해 전달된다.
* **샌드박스 탈출**: 샌드박스 내에서 생성된 결과물은 반드시 검증 스크립트를 거쳐 `root/output/`으로만 이동할 수 있다.
* **네트워크 제한**: Zone 2 환경은 화이트리스트(허용된 도메인) 외의 외부 통신을 원칙적으로 차단한다.

---

### 5. 에이전트 행동 강령 (Code of Conduct)

* **Self-Report**: 에이전트는 Zone 2 실행 전 반드시 사용자에게 실행 목적과 예상 범위를 보고해야 한다.
* **No Persistence**: Zone 2에서 발생한 임시 파일은 작업 종료 후 즉시 삭제한다. 
* **Log Integrity**: 모든 실행 로그는 호스트의 `logs/sandbox/`에 실시간으로 기록되어야 하며, 에이전트는 이 로그를 수정할 수 없다.

---

### 6. 위반 시 조치
에이전트가 정책을 위반하여 호스트 시스템에 직접적인 위험을 가할 경우(예: Zone 0에서 미검증 코드 실행), 시스템은 즉시 모든 세션을 종료하고 관리자에게 알림을 보낸다.
