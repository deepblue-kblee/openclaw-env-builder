# 🛡️ 로컬 계정 보안 및 관리 정책 (LOCAL_ACCOUNT_POLICY.md)

이 문서는 샌드박스 대신 macOS 독립 계정을 사용할 때 보안과 시스템 안정성을 유지하기 위한 운영 정책이다.

---

### 1. 계정 및 권한 관리 (Account & Privilege)
- **최소 권한 원칙**: OpenClaw 계정은 관리자(Admin) 권한이 아닌 **표준(Standard) 사용자**로 생성할 것을 권장한다.
- **Sudoers 제어**: 패키지 설치 등 필요한 경우에만 비밀번호 없이 실행 가능한 특정 명령어를 `/etc/sudoers.d/`에 정의한다.
  - *예시:* `brew`, `npm install -g` 등
- **TCC (Transparency, Consent, and Control)**: macOS 설정의 '개인정보 보호 및 보안' 탭에서 OpenClaw가 접근 가능한 폴더(데스크탑, 다운로드 등)를 명시적으로 제어한다.

### 2. 데이터 격리 (Data Isolation)
- **메인 계정 보호**: `/Users/[MainAccount]/` 디렉토리에 대한 접근은 기본적으로 차단된다.
- **공유 폴더 활용**: 메인 계정과 데이터를 주고받아야 하는 경우, `/Users/Shared/OpenClaw/`와 같이 명시적으로 생성된 공용 폴더만 사용한다.
- **민감 데이터**: OpenClaw 계정 내에서는 메인 계정의 iCloud, Keychain 등에 접근할 수 없도록 설정한다.

### 3. 네트워크 및 서비스 (Network & Services)
- **내부망 보안**: 홈 네트워크(LAN) 내에서만 동작하며, 외부 포트 포워딩은 원칙적으로 금지한다.
- **API 통신**: 외부 LLM API와의 통신은 허용하되, 비정상적인 대역폭 사용이나 대량 데이터 유출 여부를 모니터링한다.
- **방화벽**: macOS 기본 방화벽을 활성화하고, OpenClaw 관련 프로세스의 인바운드 연결을 최소화한다.

### 4. 에이전트 준수 사항 (Agent Compliance)
- 에이전트는 `/Users/` 하위의 타인 디렉토리에 접근을 시도하지 않는다.
- 시스템 전체 설정을 변경하는 명령을 내리기 전 반드시 사용자에게 확인을 받는다.
- 보안 취약점이 발견되거나 의심되는 행동이 감지되면 즉시 작업을 중단하고 보고한다.
