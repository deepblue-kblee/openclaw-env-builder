✦ 지금까지 진행한 Phase 3: 업무 층(Gemini CLI) 통합 및 샌드박스 최적화 작업 내용을 요약해 드립니다.

  1. 샌드박스 상태 점검 및 진단
   - OpenClaw 확인: jarvis-box 내부에 OpenClaw CLI가 설치되어 있음을 확인했습니다.
   - 스킬 부재: 필수 스킬(애플 메모, 리마인더, 슬랙 등)이 대부분 'missing' 상태로, 실제 자동화 업무 수행이 불가능한 상태였습니다.
   - 실패 원인 분석: npx clawhub를 통한 자동 설치가 샌드박스 보안 정책(Egress Block)으로 인해 차단(403 Forbidden)되고 있음을 확인했습니다.

  2. 하네스 엔지니어링 기반 구조 재설계 (Harness Engineering)
   - 선언적 관리 도입: 기존의 수동 명령 방식에서 탈피하여, core_harness/ 폴더 내에 환경 정의 파일을 생성했습니다.
       - openclaw-sandbox.yaml: 샌드박스 리소스, 필수 스킬 리스트, 환경 변수 정의.
       - openshell-policy.yaml: 파일 시스템 및 네트워크 접근 권한(화이트리스트) 정의.
   - 지침 최신화: GEMINI.md를 수정하여 앞으로의 모든 샌드박스 관리는 이 선언적 설정 파일을 따르도록 구조화했습니다.

  3. 환경 재구축 및 로컬 LLM 연결
   - 샌드박스 재생성: 커뮤니티 표준 openclaw 이미지를 사용하여 jarvis-box를 깨끗한 상태로 재구축했습니다.
   - 추론 엔진 연결: 호스트의 ollama-local (qwen2.5:7b) 모델을 샌드박스와 연동하여 로컬 우선(Local-First) 원칙을 실현했습니다.

  4. 현재 장애 요인 및 해결 제안
  현재 샌드박스는 "오프라인 섬" 상태입니다. NemoClaw 게이트웨이가 외부 통신을 차단하고 있어 스킬 자동 설치가 멈춰 있습니다.
