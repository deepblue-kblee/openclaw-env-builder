# 📏 도메인 문서화 표준 가이드라인 (DOCUMENTATION_STANDARDS.md)

이 문서는 AI 에이전트 환경에서 도메인 지식을 효율적으로 관리하기 위한 표준 규격이다. 모든 도메인 관련 문서는 `docs/reference/[domain-name]/` 하위에 저장되며, 본 가이드를 엄격히 준수해야 한다.

---

### 1. 파일 계층 구조 (L1 / L2 / L3)

정보의 중요도와 상세도에 따라 3단계로 분리하여 에이전트의 컨텍스트 사용량을 최적화한다.

- **L1: Index (지도)**
  - 파일명: `INDEX.md`
  - 목적: 해당 도메인의 전체 구조 파악 및 핵심 키워드 매핑.
  - 내용: 하위 문서 리스트, 주요 개념 요약, 핵심 태그.
- **L2: Core Spec (핵심 명세)**
  - 파일명: `CORE_SPEC.md`
  - 목적: '변하지 않는 기술적 진실' 정의.
  - 내용: 설치 환경, 설정 파일 스키마, 보안 제약 사항, 시스템 아키텍처.
- **L3: Deep Dive (상세 가이드)**
  - 파일명: `GUIDE_[topic].md`
  - 목적: 특정 주제에 대한 상세 설명 및 사례 연구.
  - 내용: 트러블슈팅, 단계별 튜토리얼, 확장 스킬 예시.

### 2. 파일 명명 및 저장 규칙
- **경로**: `docs/reference/[domain-name]/`
- **파일명**: 영문 소문자 및 하이픈(`-`) 사용. (예: `docs/reference/openclaw/CORE_SPEC.md`)
- **최신성**: 모든 문서는 항상 최신 버전(Latest-Only)만 유지하며, 구버전은 `archived/` 폴더로 격리한다.

### 3. 정보 추출 및 가공 원칙 (MVD & Citation)
- **MVD (Minimum Viable Documentation)**: 
  - 서술형 문장, 인사말, 배경 설명 등 노이즈를 제거한다.
  - 테이블, 코드 블록, 환경 변수 등 기술 명세를 최우선 보존한다.
- **공식 출처 표기 (Citation)**:
  - 모든 섹션 또는 문서 하단에 `Source: [URL]` 또는 `Ref: [Section ID]`를 명시한다.
  - "공식 문서 외 내용(사용자 커스텀)"은 `[Custom]` 태그를 붙여 명확히 구분한다.

### 4. 에이전트 검증 체크리스트
- [ ] L1/L2/L3 구조가 명확히 분리되었는가?
- [ ] 모든 기술 명세의 출처가 기재되었는가?
- [ ] 추측이나 임의의 설정값이 포함되지 않았는가?
- [ ] Breaking Changes 또는 보안 위험(sudo 등)이 강조되었는가?
